# Monthly Brainstorm Prep — Task Workflow

## Objective

Prepare a research-backed agenda for the monthly pillar content brainstorming session. Pull performance data, identify trends, mine the topic backlog, and present 4-6 topic recommendations for Garrett to evaluate.

## When to Run

Before the 1st of each month (or whenever the brainstorming session is scheduled).

## Required Inputs

| Input | Source | Example |
|-------|--------|---------|
| **Target month** | 2 months from now | If it's April, planning for June |
| **Previous campaign data** | ClickUp PERFORMANCE folder, ad platforms, Klaviyo | Last month's results |

## Steps

### 1. Pull Performance Data

**Ad Platforms (PPC):**
- Run `analyze-ppc-performance.md` for the last 30 days
- Note: Which campaigns had best CPL? Which keywords are converting?

**Email (Klaviyo):**
- Pull last month's campaign open/click rates
- Which email topics drove the most engagement?

**Social (Buffer):**
- Pull last month's social post performance
- Which topics/formats got the most engagement?

**ServiceTitan:**
- Pull revenue by job type for the last 90 days
- Which service categories are growing? Declining?
- Any seasonal patterns visible?

### 2. Scan for Trends and Seasonal Angles

For the target month (2 months out), consider:

**Seasonal/Calendar:**
- Fire marshal inspection cycles
- Holiday office closures
- Budget season for facilities managers (Q4 planning if targeting Q1)
- Back-to-school for universities
- Lease renewal cycles for property managers
- Construction season peaks
- Weather-related security concerns

**Industry:**
- New product launches (Medeco, access control tech)
- New service offerings Anderson Lock is rolling out
- Regulation changes affecting commercial security
- Industry news or events

### 3. Mine the Topic Backlog

Check ClickUp:
- **Content Ideas Inbox** — any ideas Garrett dropped in during the month
- **Previous brainstorming session notes** — ideas that were tabled or combined
- **Topic Backlog** — running list of future ideas

### 4. Generate 4-6 Topic Recommendations

For each recommendation, provide:

```
### Topic: [Title]

**Core message:** [1-2 sentences]
**Target audience:** [Which of the 20+ commercial segments]
**Why now:** [Seasonal angle, performance data, trend, or customer need]
**Content angles:**
- Email: [brief angle]
- Social: [brief angle]
- PPC: [keywords that align]
**Connects to previous campaigns:** [How this builds on past work]
**Confidence level:** [High/Medium/Low — based on data support]
```

### 5. Prepare the "State of Marketing" Snapshot

Brief summary to open the brainstorming session:

```
## State of Marketing — [Month]

### What Worked
- [Top 3 wins with numbers]

### What Didn't
- [Top 2-3 underperformers with numbers]

### Key Metrics
| Metric | This Month | Last Month | Trend |
|--------|-----------|------------|-------|
| Total leads | ... | ... | ↑/↓ |
| Avg CPL | ... | ... | ↑/↓ |
| Email click rate | ... | ... | ↑/↓ |
| Social engagement | ... | ... | ↑/↓ |

### Revenue Context (ServiceTitan)
- Top job types: [list]
- Growing categories: [list]
- Seasonal note: [if applicable]
```

### 6. Deliver

- Present to Garrett in the brainstorming session
- Save session notes and final topic selections to ClickUp (BRAINSTORMING > Monthly Sessions)

## Expected Output

Ready-to-present brainstorming agenda with:
1. State of Marketing snapshot
2. 4-6 topic recommendations with reasoning
3. Calendar/seasonal context for the target month

## Edge Cases

- **If ServiceTitan data pull fails,** proceed without it. Note the gap and ask Garrett for verbal input on which job types are trending.
- **If no ideas in the Topic Backlog,** that's fine — the performance data and seasonal angles should still yield good recommendations.
- **If this is the first brainstorming session,** there's no "last month" data. Focus on ServiceTitan revenue data, seasonal angles, and the AI guidelines' target audience list.
